# 📚 SwiftUI View Library
An exploration of the SwiftUI View Library and Live Preview canvas — building and browsing reusable components in Xcode.

---

## 🤔 What this is
This project demonstrates how to build a library of reusable SwiftUI views and surface them in Xcode's Library panel. It shows how to use `LibraryContentProvider` to make your custom components drag-and-droppable in the canvas, alongside built-in SwiftUI views. Great for design-system-style workflows.

## ✅ Why you'd use it
- **LibraryContentProvider** — Shows how to register custom views so they appear in Xcode's View Library (+) panel
- **Reusable components** — Demonstrates a component-first approach to building SwiftUI UIs
- **Canvas-driven development** — Makes the most of Xcode Previews and the Live Canvas for rapid iteration

## 🚀 Getting Started

### 1. Clone the Repo
```bash
git clone https://github.com/NDCSwift/SwitUIViewLibraryEx.git
cd SwitUIViewLibraryEx
```

### 2. Open in Xcode
Double-click `SwitUIViewLibrary.xcodeproj`.

### 3. Set Your Development Team
In Xcode: **TARGET → Signing & Capabilities → Team** — select your team.

### 4. Update the Bundle Identifier
Change `com.example.MyApp` to a unique reverse-domain ID.

## 🛠️ Notes
- The View Library panel (+ button) in Xcode Canvas is where custom components will appear.
- Preview must be running in Canvas mode to interact with the library.

## 📦 Requirements
- Xcode 15+
- iOS 16+
